# agent-context-ops

Reusable skill for maintaining durable repository context across `AGENTS.md`, specs, plans, and runbook-style docs.

## Guidance

- Keep the skill generic enough to reuse across repositories.
- Prefer `AGENTS.md` terminology and repository-role language over tool-specific branding.
- If examples mention paths, keep them illustrative rather than repo-exclusive.
- For plan names, prefer repo-defined conventions first; otherwise recommend sortable date-led names like `YYYY-MM-DD-topic.md`.
